V1.0版本功能框架

    主体功能1:  个人信息展示
        功能需求详解:
            1. 我的i家
                1.1 显示个人头像、手机号、用户名（用户名未设置时为用户手机号）
                1.2 提供修改个人信息的入口
                #(拓展需求,等主题搭建完成再继续进行)  
                    1.3 提供作为房客下单的查询入口
                    1.4 更换换手机号
                1.5 提供查询记录的入口,包括各中方法进行查询的方法
                1.6 提供退出登录的入口
            2. 个人信息修改
                2.1 可以修改个人头像
                2.2 可以修改用户名
                2.3 登陆手机号不能修改
                2.4 上传头像与用户名分开保存
                2.5 上传新头像后页面立即显示新头像
        步骤:
            创建数据库表,在models.py中创建相应的模型类
            通过使用views进行增删改查操作
            创建与其相对应的界面友好的web页面(其中包含界面交互)
        技术难点:
            1.进入系统时强制跳转至登录界面,与数据库数据进行验证
            2.登录成功后在主界面有个人信息栏,能够通过个人信息栏进入个人信息管理页面
            3.输入其他页面不能够登录,只能在主页进行登录,隐藏主页的登录信息,防止绕过登录进入地图主页
            4.注册和登录是否需要在同一个页面,如果分离,跳转逻辑怎么处理
            5.保持登录的方法:过滤器/cookies等,在离开页面30分钟后需要重新登录
            6.在前端页面上如果注册信息已被注册或者登录失败,直接在前端页面进行提示(红色)
        

    主体功能2:  注册/登录页面
        功能需求详解:
            1. 注册
                1.1 用户账号默认为手机号
                1.2 图片验证码正确后才能发送短信验证码
                1.3 短信验证码每60秒可发送一次
                1.4 每个条件出错时有相应错误提示
            2. 登陆
                2.1 用手机号与密码登陆
                2.2 错误时有相应提示
        步骤:
            创建数据库表,在models中创建相应的模型类
            通过views.py进行相应的增删改查操作
            创建与其相对应的界面友好的web页面(其中包含界面交互)

    主体功能3:   地图首页(1.0版本仅限于深圳地图)
        功能需求详解:
            1.1 多个房屋logo图片展示，点击可跳转至房屋详情页面
            1.2 提供登陆/注册入口，登陆后显示用户名，点击可跳转至个人中心
            1.3 用户可以选择城区、入住时间、离开时间等条件进行搜索
            1.4 城区的区域信息需动态加载
        步骤
            使用爬虫功能在大型用户网站爬取数据存储至数据表中
            包含:
                1.房屋图片缩小图
                2.房屋所在区域
                3.房源发布时间
                4.房屋价格
                5.房屋地址
                6. ...
            前端页面和后台的数据交互

    主体功能4:  房屋简要信息列表显示

            2.3 区域信息动态加载
            2.4 筛选条件更新后，页面立即刷新

            主要涉及数据库的查询操作
